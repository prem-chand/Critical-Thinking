# Plan 1: Curriculum Blueprint — Outcomes & Content Matrix

**Goal:** A student or curriculum author opening the vault can see the full 12-month curriculum's purpose, monthly progression, concept placement, and depth-level schema — the authoritative blueprint for everything that follows.

**Purpose:** Before any content is written, the curriculum's outcomes and structure must be defined, documented, and referenceable. This prevents scope drift and ensures every lesson written in Phases 2-5 maps to a declared outcome at a declared depth.

**Output:** Three documents in `Curriculum/` — Curriculum Hub (entry point), 12-Month Outcomes (Bloom's progression), Content Matrix (concept placement + frontmatter schema).

    Each month must use the format (per D-09): `"After Month N, the student can [verb] [object] by [method]."`

    Derive the verb from the Bloom's slot and the content from the thread table in CONTEXT.md:

    | Month | Outcome Statement |
    |-------|------------------|
    | M01 | "After Month 01, the student can **distinguish** arguments from non-arguments by identifying premises and conclusions in a research claim." |
    | M02 | "After Month 02, the student can **classify** formal reasoning errors by identifying validity, soundness, and common fallacies in written arguments." |
    | M03 | "After Month 03, the student can **apply** Bayes' theorem to update beliefs by computing posterior probabilities from evidence." |
    | M04 | "After Month 04, the student can **analyze** study designs by identifying confounds, sample issues, and statistical claims in a published paper." |
    | M05 | "After Month 05, the student can **evaluate** evidence quality by assessing study design, replication status, and effect size in a research finding." |
    | M06 | "After Month 06, the student can **critique** a paper's reasoning chain by detecting cognitive biases, statistical errors, and logical fallacies in a single analysis." |
    | M07 | "After Month 07, the student can **construct** causal diagrams by mapping interventions, counterfactuals, and confounders for a given research question." |
    | M08 | "After Month 08, the student can **diagnose** their own motivated reasoning by identifying identity-protective cognition and applying debiasing strategies." |
    | M09 | "After Month 09, the student can **synthesize** multiple lines of evidence by integrating logic, statistics, and scientific reasoning into an adversarial collaboration framework." |
    | M10 | "After Month 10, the student can **produce** a first complete paper audit by applying all 4 threads in a structured but imperfect evaluation." |
    | M11 | After Month 11, the student can **consolidate** their reasoning skills by revisiting weak areas and re-applying concepts from earlier months to new papers." |
    | M12 | "After Month 12, the student can **independently audit** any research paper in their field using a structured framework that evaluates claim, evidence, methodology, reasoning, and biases." |

    Note month 11 uses "consolidate" as the verb — this is intentionally outside the strict Evaluate→Create arc to reflect the valley/consolidation design (per D-08 emotional arc).     Per RESEARCH.md schema and CONTEXT.md depth-level decisions:

    ```yaml
    ---
    lesson-number: "01.03"          # String format for sortability
    month: 01                       # Integer 01-12
    week: 3                         # Integer 01-04 (per month)
    thread: Logic & Reasoning        # Controlled vocabulary (4 values)
    discipline: Philosophy/Logic     # Controlled vocabulary (6 values)
    depth-level: 1                   # Integer 1-4
    core: true                       # Boolean
    tags: [curriculum, lesson]       # Always includes 'curriculum' + type tag
    learning-objectives:             # Array of strings

    ---
    ```

    **Section 2: Depth-Level Convention** (per CONTEXT.md D-04)
    - `depth-level:: 1` — Foundation: define, recognize, identify
    - `depth-level:: 2` — Application: apply to a given paper, use in structured exercise
    - `depth-level:: 3` — Synthesis: combine multiple concepts, transfer to new context
    - `depth-level:: 4` — Teaching: explain to peer, independently audit, create framework

    4 Threads (matches Thread MOC names): Logic & Reasoning, Probability & Statistics, Scientific Reasoning, Metacognition & Calibration
